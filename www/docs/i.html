WEBSERV HTTP/1.1

A C++98-compatible HTTP web server built from scratch. Handles GET · POST · PUT · DELETE · HEAD with concurrent client connections via select() I/O multiplexing and dynamic content through CGI.

// 01   HTTP METHODS

SUPPORTED REQUESTS

GET

Retrieve a resource or collection. Does not modify server state.

200 OK
POST

Submit data to create a new resource on the server.

201 Created
PUT

Create or replace a resource at a given URI with request body.

200 OK
DELETE

Remove a target resource identified by URI from the server.

204 No Content
HEAD

Same as GET but transfers only headers, no response body.

Headers Only

// 02   ARCHITECTURE

CORE COMPONENTS

// SERVER CORE

I/O MULTIPLEXER

Manages all TCP connections using select(). Listens for incoming connections, reads requests, writes responses — all non-blocking. Supports multiple virtual servers on a single port via server_name matching.

// REQUEST PARSER

BYTE PARSER

HttpRequest class processes data byte-by-byte through a finite state machine. Handles chunked encoding, multipart form data, URI parsing, and header extraction incrementally via feed().

// RESPONSE BUILDER

RESPONSE ENGINE

Response class constructs HTTP responses: status line, headers, and body. Handles static file serving, directory listing via autoindex, and error page resolution.

// CGI HANDLER

CGI EXECUTOR

Forks child processes for .py and .sh scripts. Sets environment variables per RFC 3875, pipes stdin/stdout, and returns dynamic content to clients.
